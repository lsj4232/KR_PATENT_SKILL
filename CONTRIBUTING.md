# Contributing to KR_PATENT_SKILL

본 프로젝트는 한국 변리사 커뮤니티의 집단 지성으로 성장한다. 작은 학습 항목 하나도 환영합니다.

## 기여 방법

### 1. 학습 항목 추가 (가장 환영하는 기여)

특정 명세서 작업 중 발견한 함정·실수·좋은 패턴을 일반화하여 누적.

**좋은 학습 항목 예시**:
- ✅ "회로 발명에서 V_REF 정전위 조건은 시스템 레벨 효과로 명시할 것"
- ✅ "알고리즘 발명에서 윈도우 크기·오버랩·업데이트 주기를 변형예에 명시"
- ✅ "방법 단계 부호(S110 등)와 시스템 부호(110 등)는 같은 숫자여도 S 접두사로 명확 구분 가능"

**나쁜 예시 (너무 특수, 일반화 부족)**:
- ❌ "이번 케이스에서 V_REF=1.65V 사용"
- ❌ "도 4의 박스에서 10mV 표기 제거"
- ❌ 발명자 이름, 사건 번호, 특허법인명 포함

### 2. 표현 카탈로그 확장

`kr-patent-spec-drafting/references/ko-patent-style-rules.md`의 다음 영역에 사례 추가:
- 금지 표현 / 위험 표현 카탈로그
- 권장 표현 카탈로그
- 안티 패턴 카탈로그

### 3. 점검 항목 추가

`kr-patent-consistency-check/SKILL.md`의 마스터 리스트(A-G 카테고리)에 새 점검 항목 추가.

자동 점검(grep 가능) 항목과 수동 점검 항목을 구분하여 명시.

### 4. docx 양식 개선

`kr-patent-docx-builder/scripts/build_kr_patent.js`의 양식 상수를 수정. 변경 사유와 근거(한국 특허청 별지 양식 참조 등)를 PR 설명에 포함.

### 5. 새 스킬 추가

다음 영역의 새 스킬 환영:
- 의견서(Office Action) 응답 작성
- 거절이유 분석
- 영문 번역 (PCT 출원용)
- 우선권 주장
- 분할 출원 / 계속 출원
- 무효심판 / 정정심판 대응
- 청구항 검증 (선행기초, 인용오류, 종속관계)

새 스킬은 본 패키지의 명명 규칙(`kr-patent-*`)을 따르고, 다른 스킬과 조합 가능하도록 설계.

## PR 제출 전 체크리스트

- [ ] 학습 항목/규칙이 **특정 사건이 아닌 일반 패턴**으로 작성됨
- [ ] 변경 사유를 PR 설명에 명시 (어떤 실무 경험에서 도출됐는지, 익명화하여)
- [ ] 개인정보, 사건번호, 발명자 이름, 특허법인명, 기술 영업비밀이 포함되지 않음
- [ ] 변경 후 기존 워크플로우가 깨지지 않음 (가능하면 샘플 케이스로 테스트)
- [ ] SKILL.md의 `누적 학습 항목` 또는 `회고 노하우 누적` 섹션에 추가 시, 기존 항목과 중복되지 않음
- [ ] docx 빌더 변경 시, `validate.py`로 출력 검증

## 절대 포함하지 말 것

- **실제 사건 번호** (예: `2026-123456`, `P26031903KR-a`)
- **발명자/대리인 이름**
- **특허법인 이름**
- **기술 영업비밀**: 실제 수치, 알고리즘 파라미터, 회로 토폴로지 등
- **저작권 침해 소지가 있는 자료**: 다른 변리사의 명세서 원문 등

PR에서 발견되면 즉시 squash 또는 reject.

## 윤리 가이드

- 본 스킬들은 변리사 업무 보조 도구이며, 변리사 자격을 대체하지 않는다.
- 출원 전 자격을 갖춘 변리사의 검토가 필요함을 사용자에게 안내할 것.
- 한국 특허청의 공식 가이드라인이 변경되면 즉시 반영.

## 토론

큰 변경(새 스킬 추가, 워크플로우 변경 등)은 PR 전에 Issue로 토론을 시작하는 것을 권장.

## 감사합니다

작은 학습 항목 하나라도, 다음 사용자에게는 큰 시간 절약이 됩니다.
